JUMPER POINTS TRACKER — PRODUCT REQUIREMENTS DOCUMENT (PRD)

1.  Overview The Jumper Points Tracker is a web application that allows
    users to calculate and monitor their monthly reward points by
    analyzing on-chain activity from a wallet address across multiple
    blockchains.

The application will: - Aggregate transactions across multiple chains -
Retroactively calculate historical monthly points - Calculate current
month points in real-time - Display actions required to reach the next
reward tier

The application will be deployed on Vercel and built using React
(Next.js).

2.  Product Goals Primary Goal: Enable users to instantly understand:
    "How many Jumper points do I have, and what should I do next to earn
    more?"

Secondary Goals: - Simplify reward tracking - Encourage on-chain
activity - Provide a fast and minimal dashboard

3.  Personas Crypto Farmer:

-   Owns multiple wallets
-   Optimizes reward farming
-   Checks points monthly

Casual User: - Uses Jumper occasionally - Wants to know if close to next
tier

4.  Scope Included (MVP):

-   Wallet address input
-   Multi-chain scanning
-   Historical points calculation
-   Current month calculation
-   Recommended actions
-   Single-page minimal UI

Out of Scope (v1): - Authentication - Persistent database - Multi-wallet
dashboards - Notifications - Custom backend indexer

5.  UX / Interface Design Principles:

-   Minimal
-   Single screen
-   Fast feedback
-   Easy comprehension (<5 seconds)

User Flow: 1. User visits page 2. Inputs wallet address 3. Clicks
"Verify" 4. Application scans chains 5. Dashboard displays results

6.  Features

6.1 Wallet Input Single field: Wallet Address (0x…) [ Verify ]

Validation: - EVM address format - ENS support (optional v1.1)

6.2 Multi-Chain Scan Supported Chains (MVP): - Ethereum - Arbitrum -
Optimism - Base - Polygon - BNB Chain

Data Sources: - Public RPC endpoints - Explorer APIs fallback - Known
bridge/swap contracts

6.3 Transaction Classification Transactions categorized as: -
TRANSACTOOR: total transaction count - BRIDGOOR: bridge transactions -
SWAPOOR: swaps - CHAINOOR: number of chains used

6.4 Points Engine

TRANSACTOOR: 1–5 txs: 10 xp 5–10 txs: 18 xp 10–20 txs: 25 xp 20–35 txs:
33 xp 35–50 txs: 40 xp 50+ txs: 50 xp

BRIDGOOR: $100–1k: 10 xp $1k–10k: 18 xp $10k–50k: 25 xp $50k–100k: 33 xp
$100k–500k: 44 xp $500k+: 50 xp

SWAPOOR: Same tiers as Bridgoor.

CHAINOOR: 2 chains: 10 xp 9 chains: 30 xp max

6.5 Monthly Calculation Transactions bucketed by: timestamp → YYYY-MM

points(month) = transactoor + bridgoor + swapor + chainoor

6.6 Recommendations Display: "It is missing X actions to reach the next
tier."

Examples: - "3 more transactions → +7 XP" - "Bridge $420 more to reach
next tier" - "Use 2 new chains for maximum Chainoor"

Algorithm: nextTierRequirement - currentValue

7.  Technical Architecture

Architecture: Frontend (Next.js) | RPC Calls | Blockchain Nodes

No backend required for MVP.

Stack: - React (Next.js App Router) - TypeScript - TailwindCSS - Vercel
Deployment - viem or ethers.js - React Query for caching

Modules: 1. Chain Scanner Fetches transaction history and normalizes
data.

2.  Activity Classifier Detects swaps, bridges, and chains used.

3.  Points Engine calculatePoints(monthData) → score

4.  Recommendation Engine Compares current progress to next tiers.

5.  Performance Requirements

-   Result time under 5 seconds
-   Session caching
-   RPC pagination
-   Parallel chain requests

9.  Security

-   Read-only wallet usage
-   No signing

10. Analytics (Optional)
