<task type="auto">
  <name>Task 2: Add GitHub remote and push to repository</name>
  <files>N/A (git operations)</files>
  <action>
1. Add the GitHub remote:
   ```bash
   git remote add origin https://github.com/ttresse/jumper-watchoor.git
   ```
   Note: If remote already exists, this will error - that's fine, skip to step 2.

2. Verify remote is configured:
   ```bash
   git remote -v
   ```
   Expected: origin pointing to https://github.com/ttresse/jumper-watchoor.git

3. **Check remote repository state before pushing:**
   ```bash
   git ls-remote origin
   ```
   - If output is empty: Remote repo is empty, direct push is safe
   - If output shows refs: Remote has existing content, need to handle merge

4. Stage all files (respecting .gitignore):
   ```bash
   git add .
   ```

5. Verify sensitive files NOT staged:
   ```bash
   git status
   ```
   Confirm: .claude/, .planning/ directories should NOT appear

6. Commit the .gitignore update:
   ```bash
   git commit -m "chore: prepare for public release"
   ```

7. Push to GitHub based on remote state from step 3:

   **If remote was empty (no refs):**
   ```bash
   git push -u origin main
   ```

   **If remote has content:**
   ```bash
   git fetch origin
   git merge origin/main --allow-unrelated-histories -m "Merge remote history"
   git push -u origin main
   ```
   If merge conflicts occur, resolve them (prefer local changes for app code, keep remote README/LICENSE if they differ).

If authentication required, ensure git credentials are configured or use personal access token.
  </action>
  <verify>
