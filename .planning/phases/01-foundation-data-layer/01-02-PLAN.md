  <name>Task 1: Create useLiFiTransfers hook</name>
  <files>src/hooks/useLiFiTransfers.ts</files>
  <action>
Create new file `src/hooks/useLiFiTransfers.ts` as the main data fetching hook.

Use @tanstack/react-query's `useQuery` (NOT useInfiniteQuery - per RESEARCH.md) with internal pagination.

Implementation:

```typescript
interface UseLiFiTransfersResult {
  transfers: LiFiTransfer[];
  transactionCount: number;
  isLoading: boolean;
  isComplete: boolean;
  error: string | null;
  cancel: () => void;
  retry: () => void;
}
```

Hook logic:

1. Accept `wallet: string | null` parameter

2. Create AbortController ref for cancellation:
   - `const abortControllerRef = useRef<AbortController | null>(null);`

3. Use `useQuery` with:
   - queryKey: ['lifi-transfers', wallet]
   - queryFn: Creates new AbortController, stores in ref, calls fetchAllTransfers with signal and progress callback
   - Progress callback: calls store's updateProgress action
   - enabled: !!wallet && !isCancelled (from store)
   - retry: false (we handle retries in adapter)
   - staleTime: 5 * 60 * 1000 (5 minutes)

4. On query start: call store.startScan(wallet)
5. On query success: call store.completeScan(data)
6. On query error: call store.failScan(error.message)

7. Cancel function:
   - Call abortControllerRef.current?.abort()
   - Call store.cancelScan()
   - Call queryClient.cancelQueries(['lifi-transfers', wallet])

8. Retry function:
   - Call queryClient.resetQueries(['lifi-transfers', wallet])
   - Call store.startScan(wallet) to reset state

Return: transfers from store (empty until complete), transactionCount from store, isLoading/isComplete from query state, error from store, cancel and retry functions.

Key per CONTEXT.md:
- "Results appear only after complete load" - transfers stays [] until query succeeds
- "If cancelled, discard all data" - adapter returns [] on abort
- "Retry failed requests 2-3 times" - handled in adapter, not here
