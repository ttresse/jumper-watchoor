<task type="auto">
  <name>Task 2: Create LiFi adapter with pagination, retry, and cancellation</name>
  <files>src/adapters/lifi.adapter.ts</files>
  <action>
Create new file `src/adapters/lifi.adapter.ts` implementing the LiFi Analytics API client.

Key requirements from CONTEXT.md:
- Endpoint: `https://li.quest/v2/analytics/transfers?wallet={address}`
- Pagination via `next` cursor token
- Retry failed requests 2-3 times with exponential backoff (1s, 2s, 4s)
- Support cancellation via AbortSignal (per user constraint: cancel discards all data)
- Progress callback for live counter

Implementation:

1. `fetchWithRetry<T>(fn: () => Promise<T>, maxRetries: number = 3): Promise<T>`
   - Exponential backoff: 1000 * Math.pow(2, attempt) ms
   - Rethrow after max retries exhausted

2. `buildTransfersUrl(wallet: string, cursor?: string | null): string`
   - Base URL: `https://li.quest/v2/analytics/transfers`
   - Params: wallet, limit=100, status=DONE
   - Add cursor if present (URLSearchParams handles encoding)

3. `fetchAllTransfers(wallet: string, signal: AbortSignal, onProgress?: (count: number) => void): Promise<LiFiTransfer[]>`
   - Accumulate all transfers across pages
   - Check signal.aborted before each request and return [] if cancelled (per CONTEXT.md: discard all data)
   - Call onProgress after each page with running total
   - Use fetchWithRetry for each page request
   - Continue while response.next is not null

Handle edge cases:
- Empty wallet (no transactions): return []
- API error after retries: throw Error with status code

Do NOT: Use useInfiniteQuery (per RESEARCH.md anti-pattern), stream partial results, parallelize pages.
  </action>
