     - `loadedMonthKeys`: Set of successfully loaded months

<task type="auto">
  <name>Task 2: Create prefetch manager hook</name>
  <files>src/hooks/usePrefetchManager.ts</files>
  <action>
Create hook for background prefetching after initial load:

1. `usePrefetchManager(wallet: string | null, loadedMonthKeys: Set<string>)`:
   - Uses `useQueryClient` to access query client
   - Uses `useEffect` to trigger prefetch when:
     - wallet is set
     - loadedMonthKeys has 4 entries (initial load complete per CONTEXT.md)

2. Prefetch logic (inside effect):
   - Get all available month keys via `getAllAvailableMonthKeys()`
   - Filter out already loaded months
   - Process in batches of 3 (per CONTEXT.md: "Fetch 3 months in parallel")
   - For each batch, use `Promise.all` with `queryClient.prefetchQuery`:
     - queryKey: monthQueryKey(wallet, monthKey)
     - queryFn: () => fetchMonthTransfers(wallet, monthKey)
     - staleTime: Infinity (all prefetched months are past months)
   - Wait for batch to complete before next batch (sequential batches, parallel within)

3. Priority interruption:
   - Track prefetch state with useRef (current batch, abort flag)
   - Export `prioritizemonth(monthKey: string)` function that:
     - Uses `queryClient.ensureQueryData` for immediate fetch
     - Does NOT abort background prefetch (let it continue)
   - Per CONTEXT.md: "User requests jump the prefetch queue"

4. Error handling:
   - Silent retry via React Query defaults (3 retries)
   - Don't block on errors - continue to next month
   - Per CONTEXT.md: "Silent retry on failures"

Add 'use client' directive. Export hook.
  </action>
