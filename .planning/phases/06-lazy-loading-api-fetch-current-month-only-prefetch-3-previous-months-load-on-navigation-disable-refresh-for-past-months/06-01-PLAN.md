  <name>Task 1: Create month utility functions</name>
  <files>src/lib/month-utils.ts</files>
  <action>
Create a new module with pure functions for month-based operations:

1. `getMonthBoundaries(monthKey: string): { from: number; to: number }` - Calculate Unix timestamps for month start/end
   - Parse monthKey (YYYY-MM format)
   - Use Date.UTC for consistent UTC boundaries
   - Return `from` as start of month (00:00:00 UTC), `to` as end of month (23:59:59 UTC)
   - Both timestamps in seconds (not milliseconds) for LiFi API

2. `getCurrentMonthKey(): string` - Get current month as YYYY-MM (UTC)
   - Use Date.UTC methods to avoid timezone issues

3. `getLastNMonthKeys(n: number): string[]` - Generate array of N month keys ending with current
   - Returns chronologically sorted array (oldest first)
   - Example: getLastNMonthKeys(4) for March 2026 returns ['2025-12', '2026-01', '2026-02', '2026-03']

4. `monthQueryKey(wallet: string, monthKey: string): readonly ['lifi-transfers', string, string]` - React Query key factory
   - Consistent key structure for all month queries
   - Returns tuple as const for type safety

5. `getAllAvailableMonthKeys(): string[]` - Generate all month keys from earliest LiFi data to current
   - Start from January 2022 (LiFi launch approximate)
   - End at current month (inclusive)
   - Returns chronologically sorted array

Export all functions. Add JSDoc comments. Use UTC throughout to match existing classification.ts patterns.
  </action>
